# Column Comparison: ARR_MASTER_NEW vs MASTER_BILLING_NEW

Generated: 05/20/2026 (updated 05/20/2026: CONTRACT_LENGTH_RAW added to MBN; LENGTH_DAYS renamed to CONTRACT_LENGTH in AMN; REGION renamed to SHIP_REGION in AMN)
Supersedes: arr_master_vs_master_billing_columns.md (05/19/2026 — was based on original ARR_MASTER)

Notes:
- ⚡ = present in both tables but logic differs
- ← = column exists in MASTER_BILLING_NEW only
- → = column exists in ARR_MASTER_NEW only
- -- = column absent from that table
- MASTER_BILLING_NEW "raw column" entries are traced back to their actual
  source logic in raw_union, main, or tiers CTE, not just the final SELECT alias.
- ARR_MASTER_NEW acv formula uses product/category-aware annualization (ecomm/AlertSite/BugSnag
  branches). MASTER_BILLING_NEW uses a uniform contract_length-based annualization.
  This difference is intentional and must be preserved.
- ARR_MASTER_NEW still computes product reporting/naming/grouping fields as CASE logic in main
  (sourced from productgroup). MASTER_BILLING_NEW sources these from vw_ns_ss546_new, which
  resolves them upstream via dim_product_dm_hierarchy_tbl. Logically equivalent for NS rows.
